# 🚀 Mora AI 기능 빠른 시작

3분 안에 실제 AI 기능을 테스트하세요!

---

## ✅ 준비사항

### 1. OpenAI API 키 발급 (2분)

1. https://platform.openai.com 접속
2. 계정 생성/로그인
3. https://platform.openai.com/api-keys 에서 API 키 생성
4. 복사: `sk-proj-...`

### 2. 환경 변수 설정 (30초)

```bash
# 프로젝트 폴더에서
cd /Users/garden/Desktop/mora

# .env.local 파일 생성
echo "OPENAI_API_KEY=sk-proj-여기에-당신의-키를-붙여넣기" > .env.local
```

또는 직접 생성:

```env
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxx
```

### 3. 의존성 설치 (30초)

```bash
npm install
```

## 🎬 실행 방법

### 1. 개발 서버 시작

```bash
npm run dev
```

### 2. 브라우저 접속

```
http://localhost:3000/meeting/demo
```

### 3. 두 가지 모드 선택

#### 🎤 라이브 모드 (실제 AI)
1. **"실제 녹음 시작"** 버튼 클릭
2. 마이크 권한 허용
3. 말하기! "안녕하세요, 테스트 중입니다"
4. 5초 후 자동으로:
   - 🎯 Whisper가 음성을 텍스트로 변환
   - 🤖 GPT-4가 요약 생성
   - 📊 시각화 보드에 노드 추가

#### 🎮 데모 모드 (API 키 불필요)
1. **"데모 모드"** 버튼 클릭
2. 자동으로 한국어 회의 시뮬레이션
3. 4초마다 새 대화 추가

## 🐛 문제 해결

### ❌ "OPENAI_API_KEY가 설정되지 않았습니다"

```bash
# 1. .env.local 파일이 있는지 확인
ls -la .env.local

# 2. 내용 확인
cat .env.local

# 3. 서버 재시작
npm run dev
```

### ❌ "마이크 접근 권한이 필요합니다"

Chrome:
1. 주소창 왼쪽 자물쇠 아이콘 클릭
2. "사이트 설정"
3. 마이크 → "허용"
4. 페이지 새로고침

### ❌ 소리가 안 나오는데 녹음이 안 됨

- 마이크가 제대로 연결되어 있는지 확인
- 시스템 설정에서 마이크 권한 확인
- 브라우저 콘솔(F12) 확인

### 최적의 테스트 환경

1. **조용한 장소**: 배경 소음 최소화
